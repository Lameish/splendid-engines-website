# Splendid Engines Visual Brand

Last updated: 2026-05-12. Locked outputs from the Round-2 mockup review with Evan and Brandon, plus the deck framework locked in the follow-up session. This skill operationalizes the April 2026 Production Brand Guide and the May 2026 Visual Framework + Deck Framework into rules Claude can apply on any visual task.

**Two surfaces, one brand.** The shared rules (logo, palette, fonts, Daffodil rule, iconography, imagery, retire list) apply everywhere. The layout system differs by surface: **web uses Hybrid (§3)**, **decks use Light with hard-truth exceptions (§4)**. When asked to make something, check which surface the task is for, then apply the shared rules plus the right layout system.

When this skill conflicts with the April 2026 Production Brand Guide, this skill wins. When this skill is silent on a detail, defer to the Production Brand Guide. When both are silent, flag the gap rather than inventing an answer.

---

## 1. The Thesis

The Splendid brand sits at the intersection of two ideas: **Splendid** (floral, handcrafted, human) and **Engines** (mechanical, systematic, precise).

The logo carries the warmth. The system carries the engine.

This means: the logo is allowed to be hand-drawn, textured, and a little bit literary. Everything else in the visual system — typography, layout, imagery, color usage — is structured, deliberate, and quiet. The warmth is concentrated in the logo. The engine does the rest.

The buyer is a senior B2B revenue leader (CRO archetype, Dennis profile). Time-poor, billion-dollar credentials, has seen everything. They need to look at a Splendid surface and immediately read it as the work of two senior operators running a real operation. Not a creative agency. Not a wellness studio. Not a corporate consultancy either. Editorial-prestige register: New Yorker, not LinkedIn-influencer.

When in doubt: lean cleaner, leaner, more deliberate. Never lean corporate. Never lean cute.

## 2. What Is Locked (Do Not Change)

These rules apply on every surface — web, deck, social, email, product UI, print.

### Logo
The Splendid Engines stamp logo is locked. Cursive "Splendid" + slab uppercase "ENGINES" + flower and wrench. Hand-carved-stamp texture. This is the entire warmth quota for the brand outside of occasional ornamental moments.

The mark works in three production color treatments:
- **White on Lagoon** — primary use: hero, proof band, testimonial bookend, dark merch, dark slides
- **Lagoon on Snow** — light surfaces: nav, footer, business cards, light merch, print, light slides
- **White on Poppy** — special arrival moments: stickers, hats, sparing merch

Brandon is the source of the production logo files (production-ready transparent PNGs in all three treatments).

### Color Palette
From the April 2026 Production Brand Guide. All colors stay exactly as specified. The elevation is in **usage discipline**, not in changing the colors.

| Name | Hex | Role |
|---|---|---|
| Lagoon | `#153047` | Primary text on light. Hero/proof/testimonial backgrounds on web. Hard-truth slides only on decks. The dark-mode workhorse. |
| Poppy | `#DB4937` | Signal accent. CTAs. Eyebrows. One emphasized word per layout. Carries accent role on dark surfaces (where Daffodil is forbidden). |
| Lake | `#377BDB` | Secondary accent. Links. Data viz support. Never a primary background. |
| Daffodil | `#F2ED44` | Highlighter rule (see §5). Stat units on dark. Never a background. |
| Maté | `#575C45` | Atmospheric panels and dividers only. Never a primary content background. |
| Snow | `#FFFFFF` | Primary background on light surfaces. The default on decks. |
| Ash | `#EBF4F7` | Secondary light background. Subtle bands. Cards on Snow. Section dividers on decks. |
| Ash Warm | `#F4ECDF` | Warm light background. "Who we help" and similar editorial sections. |

### Fonts
From the Production Brand Guide:
- **Display / Heading**: Hanley Pro Slim Bold, +25 to +40 tracking, uppercase, line-height 1.0–1.05
- **Eyebrow / kicker**: Hanley Pro Slim Bold, +120 to +180 tracking, 13px web / 11–13pt deck, uppercase, Poppy color
- **Body**: Brandon Grotesque Regular, 17–18px web / 18–22pt deck / 11–12pt print, line-height 1.55–1.65 web, 1.4–1.5 deck
- **Special-occasion**: Hanley Monoline cut for rare hand-drawn moments (signature line, single pull quote, footer signoff). Not a heading face.

Both faces are commercial licenses. When rendering in environments without licensed access (mockups, web previews, internal artifacts), use these substitutes and call out the substitution explicitly:
- Hanley Pro -> Big Shoulders Display (Google Fonts)
- Brandon Grotesque -> Mulish (Google Fonts)
- Hanley Monoline / script accents -> Caveat (Google Fonts)

The substitutes are for layout-and-spacing review only. Production assets must use the licensed fonts.

### Tagline
"Nice day, isn't it?" — with the question mark. It is a question.

### Internal copy convention
Splendid uses the word **operators**, not "practitioners," to describe Evan and Brandon. "Senior operators." "Two senior operators." "Operator-grade." Match this in any new copy.

## 3. The Hybrid Layout System (Web)

**Use this for any work where the surface is the marketing website, a landing page, or a long-form scrolling web layout.** Sub-pages may use the Full Light variation (see below). Decks use the Light system in §4 instead.

The homepage and any page-level web layout uses the **hybrid** treatment:

- **Above the fold loads dark.** Lagoon hero. White headline. Poppy accent. The dark surface earns gravity for the brand statement.
- **As the user scrolls, the palette shifts to light.** Snow / Ash / Ash Warm dominate from the proof band onward.
- **Dark surfaces return only for moments that earn them.** The "delivering hard truths" insight: proof numbers, the testimonial. These benefit from the same gravity as the hero.
- **Closing testimonial is allowed to live on Lagoon** as a bookend even when the rest of the page is light.
- **Footer is light.**

### When to use Full Light (web sub-pages)

Sub-pages where gravity isn't the first job: blog posts, case studies, methodology deep-dives, pricing breakdowns. Light surfaces throughout. Same fonts, same accent rules.

### When to use Full Dark (rare, specific cases)

Brochure covers. Single-page PDF cover pages used as standalone artifacts. Premium positioning materials where the entire surface is meant to read as serious from edge to edge. Never as the primary register for the marketing site. Never as the default for slide decks (see §4 — decks use Light by default).

### Claim-band rule (the proof numbers on the homepage)

The proof-numbers band on the homepage is always:
- On Lagoon (dark)
- Stats centered within their columns, not left-aligned
- Numbers in white with Daffodil units (% / + / etc.)
- **Logo salad above the numbers**, not below. Client logos earn the right to claim the numbers.
- Client logos rendered on a Snow band so brand colors stay legible. If color logos won't be available, fall back to white silhouettes on Lagoon.

## 4. The Light Layout System (Decks)

**Use this for any work where the surface is a slide deck. The build tool is Google Slides, always.** Canva may produce individual images that get dropped into Slides; the deck itself lives in Slides. Three deck types: sales decks, reports, pitch decks. Same system, different patterns per type.

### The default is light

Every slide defaults to Snow. Lagoon (dark) is reserved for **hard-truth slides only**:
- **Proof slides** — one big number (Hanley 120–180pt) on Lagoon. Centered. No body copy.
- **Diagnostic slides** — the hard truth about the buyer's situation. Eyebrow + headline + short body, on Lagoon.
- **Testimonial slides** — client quote on Lagoon, headshot above the quote (not beside), attribution as eyebrow underneath.

### Maximum Lagoon ratio

No more than **1 in 5 slides** on Lagoon. A 15-slide sales deck gets 2–3 Lagoon slides total. A 30-slide report gets 4–6. Beyond that ratio, the dark surface stops signaling weight and starts feeling like the wallpaper of the deck.

### The rule of thumb

If you cannot answer the question *"why does this slide deserve gravity?"* with a one-sentence reason, the slide does not get Lagoon. Default to Snow.

### Three deck types

| Type | Length | Cover | Mode | Notes |
|---|---|---|---|---|
| **Sales deck** | 10–15 slides | May be Lagoon (one-time gravity moment) | Presented live | Brand-heavy. Proof slide always Lagoon. Testimonial always Lagoon. Methodology/fit/operators: light. One Daffodil highlight per deck max. |
| **Report** | 20–40 slides | Light by default | Consumed as PDF | Text-heavy editorial register. Section dividers on Ash or Lagoon for chapter breaks. Charts and tables get whole slides. |
| **Pitch deck** | 15–25 slides | Light, restrained | Formal | Less brand showmanship. Heavy use of proof slides. Daffodil minimal — once or twice across the whole deck. Default to 2-column layouts. |

### The 10-foot rule

Deck typography must work at distance — laptop on a conference table, projected on a wall, screen-shared on a 13" Zoom window. **Minimum body 18pt. Slide titles 40–48pt. Big stat numbers 120–180pt.** The 10-foot test: if you stand 10 feet from your screen and can't read the body copy, the type is too small.

### Deck type scale (Google Slides point sizes)

| Role | Font | Size | Use |
|---|---|---|---|
| Slide title (H1) | Hanley Slim Bold, +25 | 40–48pt | The single dominant line on most slides. Uppercase. |
| Cover title | Hanley Slim Bold, +25 | 60–72pt | Cover slide only. Larger than any content slide. |
| Subhead (H2) | Hanley Slim Bold, +25 | 28–32pt | Secondary line on titles that need two-tier hierarchy. |
| Eyebrow / section marker | Hanley Slim Bold, +180 | 11–13pt | Poppy color. Above every slide title. |
| Body copy | Brandon Grotesque Regular | 18–22pt | Minimum 18pt. Never smaller. Line-height 1.4–1.5. |
| Big stat number | Hanley Slim Bold | 120–180pt | The hero number on proof slides. Bigger than feels comfortable. |
| Caption / footnote | Brandon Grotesque Regular | 12–14pt | True footnotes only. Never primary body content. |
| Footer / page number | Hanley Slim Bold, +120 | 9–10pt | Logo lockup left, page number right. |

### Deck-specific Daffodil ceilings

The §5 Daffodil rule applies, but with tighter ceilings for decks: **one Daffodil highlight per slide maximum. Three highlights across the whole deck maximum** — not three per section, three total. If Daffodil is on every other slide, the eye stops registering it as emphasis.

### The Google Slides slide master

Configure these six layouts in the Slides master (`View -> Theme builder`). Every slide uses one of these as its base:

| Layout | Surface | Use |
|---|---|---|
| Cover | Lagoon (sales) or Snow (report, pitch) | First slide. Deck title in Hanley 60–72pt. |
| Section divider | Ash (or Lagoon for major chapter breaks) | "01 / Methodology" eyebrow + section name in Hanley 48pt. |
| Content (default) | Snow | Eyebrow + title (40–48pt) + body or 2-column grid. The workhorse layout. |
| Proof slide | Lagoon | One big number (120–180pt) + label. Centered. No body copy. |
| Testimonial | Lagoon | Eyebrow + quote (22–28pt) + attribution. Headshot above quote. |
| Closing CTA | Snow (default) or Lagoon (sales decks only) | Headline + single Poppy CTA button + tagline in script. |

**Page setup:** 16:9 widescreen, 10in × 5.625in. Top and bottom margins 0.5in. Left and right margins 0.6in. Content area 8.8in × 4.625in. Footer reserve: bottom 0.4in for logo lockup and page number.

**Footer lockup on every slide:** Logo bottom-left (0.4in from left, 0.25in from bottom, 0.3in tall). Page number bottom-right (0.4in from right, 0.3in from bottom, Hanley 9pt with +120 tracking, 55% opacity).

### One file, many decks

Build a single "Splendid Master Deck" in Google Slides with all six layouts configured. To make a new deck, duplicate the master file and rename. Anyone editing the deck can only use the layouts that exist; the system maintains itself. Update the master file when the brand evolves; cascade updates to derivative decks via Slides' theme system.

### Imagery on slides

Decks have less room for imagery than the web. The hero illustration that anchors a homepage doesn't belong on every slide.

**What appears on slides:**
- The Splendid logo lockup, small, bottom-left of every slide (footer)
- Animal totems with people on the operators slide and "who we are" slide
- The flagship hero illustration (sociogram) on the cover slide of a sales deck or opening of a report — one per deck, no more
- Charts and data visualizations built in the brand palette: Lagoon primary, Poppy and Lake as accents
- Client logos in their official wordmarks on the "we've put our clients in the room with" slide

**What does NOT appear on slides:**
- Stock photography of generic business people, handshakes, lightbulbs, or office scenes
- Decorative illustrations from the retire list (§10)
- Background images behind body text — type sits on solid Snow or solid Lagoon
- Photo collages or moodboards as content slides
- Animated GIFs or video (unless the deck is digital-only and motion does specific work)

**The number is the asset.** If a slide needs to communicate "deal size," show the number itself in Hanley at 120–180pt on Lagoon. The number is the imagery. We don't decorate claims with stock illustrations of charts going up.

## 5. The Daffodil Highlighter Rule

**Applies to both web and decks.** Daffodil yellow gets one job: a highlighter mark under a single load-bearing word in a display headline.

**Rules:**
- **Light surfaces only.** On Lagoon, Daffodil reads as unpleasant clash. Never highlight on dark.
- **One word per headline.** Never two. The highlight transforms a noun into a claim — multiple highlights dilute the claim.
- **Web ceiling:** two highlights per page maximum. Hero and one other section.
- **Deck ceiling:** one highlight per slide maximum, three across the whole deck maximum (tighter — see §4).
- **Word must fit on a single line.** If the highlight wraps across line breaks, the rendered effect is two disconnected rectangles instead of one continuous emphasis. Either rewrite the headline so the highlighted phrase fits on one line, or move the highlight to a different word.
- **CSS implementation**: `background: linear-gradient(to top, #F2ED44 12%, #F2ED44 42%, transparent 42%); padding: 0 0.04em; box-decoration-break: clone;`
- **The highlight occupies roughly 38–42% of cap height**, sitting on the baseline. Not a full underline (too thin) and not a full block fill (too dominant).

**On dark surfaces**, the equivalent emphasis role goes to **Poppy color**, applied to a single word as text color (not a highlight bar). Example: "Not you. **Yet.**" — "Yet." is set in Poppy.

## 6. Iconography

**Applies to both web and decks.** Same icon system, scaled appropriately per surface.

### Locked conventions
- **Stroke weight**: 2.5px at 24px viewbox (scales proportionally — 3.5px at 32–40px, 5px at 72–96px, 8–10px at 200px+)
- **Caps**: square. Never rounded.
- **Joins**: miter. Never round or bevel.
- **Style**: outline. No fills except for endpoint dots (target node, signal source).
- **Color**: Lagoon by default on light surfaces, Poppy when used as a methodology accent, white on dark. Never Daffodil.
- **Standard sizes web**: 24px (inline) and 48px (section markers).
- **Standard sizes deck**: 32–40px (section marker next to eyebrow), 72–96px (3-up icon grid), 200px+ (hero icon, single icon as primary visual).
- **In Google Slides specifically**: insert icons as SVG via `Insert -> Image -> Upload from computer`. Never rasterized PNG — they pixelate when projected. To recolor, swap the source SVG's stroke color in a text editor and re-import.

### Building a new icon
1. Start from a domain-model object or a real conceptual primitive. Don't invent decorative icons.
2. Construct on a 24px grid with 2px padding from edges.
3. Stroke once at 2.5px, square caps, miter joins.
4. Test at 24px and 48px. If it doesn't read at 24px, simplify. If geometry breaks at 48px, redraw.
5. Render in Lagoon and white. If it works in both, ship.

### v1 set used in the homepage mockup and deck framework
- Relationship / network — five-node cluster with central connection
- Signal — concentric arcs radiating from a base dot
- Engine — gear-with-rays geometry
- Conversation — two overlapping speech-bubbles
- Pipeline — chevron-bracketed flow form
- Discovery — magnifier with crosshair
- Arrow right — horizontal line + chevron
- Check — three-stroke checkmark

These are the v1 set. The full library extends from the Splendid Domain Model and is being built as a separate effort.

### Coordinating with product UI
The icon system must extend into product UI when the Glide-to-Supabase migration ships. Same line weight, same caps, same joins. The product and the marketing site and the decks use one icon language. If a marketing icon and a product icon represent the same domain object, they are the same icon.

## 7. Imagery

**Applies to both web and decks.** See §4 for deck-specific imagery rules (less imagery on slides than on web, "the number is the asset").

### Locked direction
**The illustration vocabulary is hand-carved stamp.** The logo is a stamp. Animal totems are stamps. That's the unifying texture across all hand-drawn elements.

### What's in
- **Animal totems with people**, in the hand-carved-stamp register. Wolf, elk, hawk, bear, ram, similar megafauna or apex-predator subjects. Always paired with a specific person — partner photo, client headshot, expert profile. Never floating on a section by itself as decoration.
- **Editorial photography of the buyer's peer group**. People who could be on a CRO's leadership team. Mid-30s to mid-50s, business contexts, quiet portraits. Diverse but plausible. No lifestyle shoots, no jumping figures, no party shots.
- **Numbers as imagery.** "50–60%" in Hanley at 96px on Lagoon is a stronger visual than any chart. Treat headline numbers as the visual feature of the section/slide they live on.
- **The flagship hero illustration: a sociogram**. A hand-drawn network diagram showing buyer-side relationships of varying strength, with one buyer node highlighted. Same hand-carved-stamp register as the logo. This is the primary hero image for the website homepage and the sales deck cover slide. Brandon is the source — built in Canva or commissioned.

### What's out (retire list)
- **Bumblebees**, dancing-figure sketches, money-roll sketches, typewriter sketches, and similar zine-style illustrations from the Cabriah-era site. Same illustration technique as the wolf and elk, but the wrong subject matter — they read as boutique creative agency.
- **Stock photo collages with random faces.** The "Community influence" hero on the current site (orange-haired guy, casual portraits, Swiss army knife clip-art on Maté canvas) is the canonical example of what not to make.
- **Torn-paper edges, frayed paper textures, parchment backings.** Gone from any new design. The hand-drawn elements stand on their own; they don't need a torn-paper costume.
- **Hand-drawn dividers and ornamental flourishes between sections.** Section transitions are whitespace, a 1px Silver line, or a Lagoon-to-Snow color change. Not torn paper.

### Partner photo treatment
The intended composition for partner portraits (Evan, Brandon, future operators):
1. **Background-removed photo cutout.** Face and shoulders only, transparent PNG. Floats against the page.
2. **Animal totem-stamp peeking from one side**, drawn separately as a standalone hand-carved-stamp illustration. Right side for Evan's elk. Left side for Brandon's wolf.
3. **No containing card, no torn-paper frame, no brick wall.** Partner and totem are the only elements.
4. The totem is ornamental — sized smaller than the partner, peeking in rather than dominating.

When the cutouts and dedicated totems aren't yet available, fall back to the existing site photos with a vignette mask that fades the photo edges into the page background. Add a clear note that production needs the proper assets. Never ship the vignette fallback as a finished design.

### The Wink
A small number of hand-drawn moments are allowed across a Splendid surface as "the wink" — little flashes of the maker's hand showing through an otherwise clean editorial system.

Examples: a sketched leaf flourish in negative space, the script "Nice day, isn't it?" in the footer, the cursive "Splendid" in the logo. These should feel like skin showing — sparing, intentional, never the dominant element on a surface. Two or three winks across an entire homepage is the ceiling. One per deck.

## 8. Layout Principles

**Applies to both surfaces.** Web-specific spacing in §3; deck-specific spacing and slide master in §4.

### Editorial structure, no decoration

- **Generous whitespace.** Section padding minimum 80px top/bottom on the marketing site. Slide margins 0.5–0.6in on decks. Editorial publications and senior B2B firms both share one trait: they let the content breathe.
- **Single-column dominant on long-form, two-column on landing.** Blog posts and case studies: single column, 65–70ch measure. Homepage and landing pages: hero is single column; supporting sections may split into 2 columns. Three columns is rare; never use a 3-column layout for headline-and-icon arrangements. Same rule on decks: 2 columns max.
- **Eyebrows above headings, every time.** Hanley Slim Bold, +180 tracking, 13px web / 11–13pt deck, Poppy. "01 · Methodology." "Section 03." "What We Do." They orient the reader and give every section/slide a structural label.
- **Quotes earn their own surface.** Client testimonials get their own section/slide, on Lagoon or Ash, with the quote at 22–28px Brandon Grotesque, attribution in Hanley eyebrow style underneath. Headshot above the quote, not beside it.
- **Numbers get full-bleed treatment.** Stats are the visual feature of the section/slide they live in. Centered within columns, not left-aligned. No accompanying illustration.

### The Hanley + Brandon hierarchy
- **Never set body copy in Hanley.** Hanley is display only.
- **Never set headings in Brandon.** Brandon is body only.
- **One display weight on screen at a time.** Don't stack three sizes of Hanley on a page/slide. One H1, one H2, one eyebrow.
- **Generous line-height on body**: 1.55–1.65 web, 1.4–1.5 deck. Every Splendid surface has more breathing room than the reader expects.

## 9. Decision Heuristics for Common Visual Tasks

When asked to make X for Splendid, default to these patterns:

### Website page or section
- Apply the Hybrid system (§3). Above the fold dark, body light, testimonial bookend optional.
- Sub-pages can use Full Light if gravity isn't the first job.
- Two Daffodil highlights per page max.
- One Poppy CTA per page.
- Sticky nav, color-shifts from dark to light on scroll on the homepage.

### Sales deck (Google Slides)
- See §4 for full deck system. Light by default, Lagoon for the 2–3 hard-truth slides.
- 10–15 slides. Cover may be Lagoon (one-time gravity moment). Closing CTA in Poppy.
- One Daffodil highlight across the entire deck.
- Use the Splendid Master Deck file. Duplicate it; build from layouts; don't invent new ones.

### Report (Google Slides)
- See §4. Light system, 20–40 slides.
- Cover light, restrained — reports get read alone, not presented.
- Section dividers every 5–7 slides (Ash or Lagoon).
- Add an executive summary slide (1 slide, 4–6 bullets, body 18pt+).
- Charts and tables get whole slides.

### Pitch deck (Google Slides)
- See §4. Light system, 15–25 slides.
- More restrained than a sales deck. No animated transitions. No clever fonts.
- Daffodil highlights twice maximum across the whole deck.
- Heavy use of proof slides — numbers earn credibility with this audience.
- Final slide is a contact slide (names, emails, calendar link), not a CTA.

### Social imagery (LinkedIn, Twitter, etc.)
- Lagoon background, white Hanley headline, Poppy or Daffodil accent on one word (Daffodil only if there's enough light surface around it; otherwise Poppy).
- The Splendid logo small in a corner.
- One animal totem or sociogram element if the post needs visual weight; otherwise text-only is fine.
- Body text in Brandon Grotesque, 18–20px, white on Lagoon.
- Avoid: emoji, gradients, photographic backgrounds, multi-color treatments.

### One-page document (PDF, printable)
- Snow background, Lagoon body text, Hanley headlines.
- Eyebrow above every section.
- Daffodil highlighter on one or two key words across the whole document.
- Footer with logo and "Nice day, isn't it?" tagline.

### Email design
- Light background (Snow), Lagoon body text.
- Hanley headlines, Brandon body.
- Single Poppy CTA button.
- No animal illustrations in email; keep it clean.
- Logo in header, small.

### Product UI (when this becomes relevant)
- Light backgrounds dominant; reserve Lagoon for navigation surfaces and modal headers.
- Iconography per §6. Lagoon or Poppy stroke. Never Daffodil.
- Same Hanley + Brandon font pairing.
- Daffodil reserved for highlight states (selected row, active filter), not a primary action color.
- Poppy is the only primary action color.

## 10. The Retire List (Specific Things to Never Bring Back)

Concrete elements that are out of the system. If you're tempted to reach for any of these in a new design, stop and choose differently.

### From the current splendidengines.com site
1. The "Community influence" hero collage with stock-photo faces and Swiss army knife clip-art
2. The three bee illustrations ("Connect / Build / Fill")
3. The "Are you ready" sketches (money-roll, typewriter, dancing-people)
4. The footer rabbit illustration
5. The torn-paper/frayed-edge backing on partner photos
6. Default HubSpot system fonts (Roboto-ish) in place of the Hanley + Brandon spec
7. "Follow us on Facebook" link going to LinkedIn (rename to "Connect on LinkedIn")
8. Decorative hand-drawn dividers between sections
9. Maté green as a primary content background
10. Three-column icon-and-text grids (use two columns or rethink the section)

### Deck-specific
11. **Default Google Slides themes.** Any built-in template with default fonts (Arial, Roboto, etc.). Always start from the Splendid Master Deck.
12. **Body copy smaller than 18pt on slides.** Violates the 10-foot rule. If a slide needs more than 40 words, split it into two.
13. **Stock photography on slides.** Handshakes, lightbulbs, office scenes. Use a number, icon, or quote instead.
14. **Animated transitions and slide builds.** No flying-in bullets, no fade-and-zoom, no carousel transitions. Cuts are clean. Exception: a single intentional reveal on a proof slide, used sparingly.
15. **Footer with company name + date + copyright on every slide.** Just logo lockup and page number. The deck is the artifact; the footer doesn't need to remind the audience whose deck it is on every page.
16. **Daffodil on Lagoon slides.** Same rule as web. Daffodil clashes on dark. Poppy carries the accent role instead.
17. **Background images behind body text.** Type sits on solid Snow or solid Lagoon. If a slide has imagery, it's the primary content, not a decoration behind type.

## 11. When Undecided

When this skill is silent on a detail and the work needs an answer:

1. **First**: check whether the question is about web layout (§3) or deck layout (§4) and read the relevant section. Most surface-specific questions have answers there.
2. **Second**: check the April 2026 Production Brand Guide in Drive. It covers detailed typography specs, color hex values, and base usage rules.
3. **Third**: defer to the operating principle — lean cleaner, more editorial, more deliberate. Never reach for ornament. Never lean corporate.
4. **Fourth**: if the question is genuinely undecided (e.g., "what's the right treatment for podcast cover art?" — never discussed), flag the gap explicitly. Surface it to Evan or Brandon as a decision they need to make, rather than inventing an answer that becomes load-bearing later.

The skill is a living document. When new decisions get locked, update this file. Date the change at the top.
